# Pricing Update Tracker

Use this file to track which sets still need a pricing refresh.

How to use:
- Mark a set as `[x]` after you run the scraper and confirm the pricing output looks right.
- Switch it back to `[ ]` if you want to re-run a set later.
- Add a date or note beside a set if something failed or needs review.

Example command:

```bash
npm run scrape:pricing -- --set=sv01
```
